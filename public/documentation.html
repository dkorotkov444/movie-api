Movie API Developers' Documentation

This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Feature Requirements

Essential Features

  • Return a list of ALL movies to the user

    Description: Returns a list of all movies available in the database.

    Method Type: GET

    Endpoint URL: /movies

    Data returned: An array of movie objects.

  • Return a list of ALL users to the user

    Description: Returns a list of all users in the database.

    Method Type: GET

    Endpoint URL: /users

    Data returned: An array of user objects.

  • Return data about a single movie by title to the user

    Description: Returns a single movie object by its title.

    Method Type: GET

    Endpoint URL: /movies/{title}

    Query Parameters: title: The title of the movie to retrieve.

    Data returned: A movie object.

  • Return data about a genre by name

    Description: Returns a genre object with its name and description.

    Method Type: GET

    Endpoint URL: /movies/genres/{genreName}

    Query Parameters: genreName: The name of the genre (e.g., 'Sci-Fi', 'Crime').

    Data returned: A genre object.

  • Return data about a director by name

    Description: Returns a director object with their bio and birth year.

    Method Type: GET

    Endpoint URL: /movies/directors/{directorName}

    Query Parameters: directorName: The name of the director (e.g., 'Quentin Tarantino').

    Data returned: A director object.

  • Allow new users to register

    Description: Registers a new user account with a provided username.

    Method Type: POST

    Endpoint URL: /users

    Data returned: A string indicating successful registration.

    Request Body: Expected fields: { "Username": "string" }

  • Allow users to update their user info

    Description: Updates the username of an existing user.

    Method Type: PATCH

    Endpoint URL: /users/{username}

    Query Parameters: username: The current username to update.

    Data returned: A string confirming the update.

    Request Body: Expected fields: { "newUsername": "string" }

  • Allow users to add a movie to their list of favorites

    Description: Adds a movie to a user's list of favorite movies.

    Method Type: POST

    Endpoint URL: /users/{username}/{movieTitle}

    Query Parameters: username: The username of the account. movieTitle: The title of the movie.

    Data returned: A string confirming the action.

  • Allow users to remove a movie from their list of favorites

    Description: Removes a movie from a user's list of favorite movies.

    Method Type: DELETE

    Endpoint URL: /users/{username}/{movieTitle}

    Query Parameters: username: The username of the account. movieTitle: The title of the movie.

    Data returned: A string confirming the action.

  • Allow existing users to deregister

    Description: Deregisters (deletes) a user from the database.

    Method Type: DELETE

    Endpoint URL: /users/{username}

    Query Parameters: username: The username to deregister.

    Data returned: A string confirming the deletion.

Optional Features

  • Allow users to see which actors star in which movies

    Description: Returns a list of all actors starring in a specific movie.

    Method Type: GET

    Endpoint URL: /movies/{title}/starring

    Query Parameters: title: The title of the movie.

    Data returned: An object containing the array of actors.

  • Allow users to view information about different actors

    Description: Returns information about a specific actor. (Endpoint exists, but logic is not yet implemented.)

    Method Type: GET

    Endpoint URL: /movies/actors/{actorName}

    Query Parameters: actorName: The name of the actor.

    Data returned: A success message.

  • Allow users to view more information about different movies

    Description: Returns more detailed information about a specific movie, such as rating and release date. (Endpoint exists, but logic is not yet implemented.)

    Method Type: GET

    Endpoint URL: /movies/{title}/full-info

    Query Parameters: title: The title of the movie.

    Data returned: A success message.

  • Allow users to create a “To Watch” list in addition to their “Favorite Movies” list

    Description: Adds a movie to a user's 'to-watch' list. (Endpoint exists, but logic is not yet implemented.)

    Method Type: POST

    Endpoint URL: /users/{username}/{to-watch}

    Query Parameters: username: The username of the account. to-watch: The title of the movie to add to the watch list.

    Data returned: A success message.

Technical Requirements

  • The API must be a Node.js and Express application.
  • The API must use REST architecture, with URL endpoints corresponding to the data operations listed above.
  • The API must use at least three middleware modules, such as the body-parser package for reading data from requests and morgan for logging.
  • The API must use a “package.json” file.
  • The database must be built using MongoDB.
  • The business logic must be modeled with Mongoose.
  • The API must provide movie information in JSON format.
  • The JavaScript code must be error-free.
  • The API must be tested in Postman.
  • The API must include user authentication and authorization code.
  • The API must include data validation logic.
  • The API must meet data security regulations.
  • The API source code must be deployed to a publicly accessible platform like GitHub.
  • The API must be deployed to Heroku.
