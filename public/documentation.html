Movie API Developers' Documentation

This web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Version: 1.1.0 | Last Updated: September 23, 2025

API Endpoints

Essential Features

  • Return a list of ALL movies (titles only)

    Description: Returns a list of all movies available in the database.

    Method Type: GET

    Endpoint URL: /movies

    Data returned: An array of movie titles (strings).

    
[
  "Reservoir Dogs",
  "The Godfather Part II",
  "Predator",
  "Zombeavers",
  "Pulp Fiction",
  "Goodfellas",
  "The Departed"
]

  • Return complete information about all movies

    Description: Returns a list of all movie objects in the database with their full details.

    Method Type: GET

    Endpoint URL: /movies/complete

    Data returned: An array of movie objects.

    
[
  {
    "_id": "68cd8259d8540b738fcebea9",
    "title": "Reservoir Dogs",
    "description": "...",
    "release_year": 1992,
    "image_url": "...",
    "rating_imdb": 8.3,
    "featured": true,
    "starring": ["Harvey Keitel", "Tim Roth"],
    "director": { "name": "Quentin Tarantino", "bio": "...", "birth_date": "1963-03-27T00:00:00.000Z" },
    "genre": { "name": "Crime", "description": "..." }
  },
  ...
]

  • Return data about a single movie by title to the user

    Description: Returns a single movie object by its title.

    Method Type: GET

    Endpoint URL: /movies/{title}

    Path Parameters: title: The title of the movie to retrieve.

    Data returned: A movie object.

    
{
  "_id": "68cd8259d8540b738fcebea9",
  "title": "Reservoir Dogs",
  "description": "...",
  "release_year": 1992,
  "image_url": "...",
  "rating_imdb": 8.3,
  "featured": true,
  "starring": ["Harvey Keitel", "Tim Roth"],
  "director": { "name": "Quentin Tarantino", "bio": "...", "birth_date": "1963-03-27T00:00:00.000Z" },
  "genre": { "name": "Crime", "description": "..." }
}

  • Return data about a genre by name

    Description: Returns a genre object with its name and description based on the provided genre name.

    Method Type: GET

    Endpoint URL: /movies/genres/{genreName}

    Path Parameters: genreName: The name of the genre (e.g., 'Sci-Fi', 'Crime').

    Data returned: A genre object.

    
{
  "name": "Crime",
  "description": "Crime film is a film belonging to the crime fiction genre. Films of this genre generally involve various aspects of crime."
}

  • Return data about a director by name

    Description: Returns a director object with their name, bio, and birth date based on the provided director name.

    Method Type: GET

    Endpoint URL: /movies/directors/{directorName}

    Path Parameters: directorName: The name of the director (e.g., 'Quentin Tarantino').

    Data returned: A director object.

    
{
  "name": "Quentin Tarantino",
  "bio": "Quentin Jerome Tarantino is an American filmmaker, actor, and author...",
  "birth_date": "1963-03-27T00:00:00.000Z"
}

  • Return a list of all users

    Description: Returns a list of all user objects in the database.

    Method Type: GET

    Endpoint URL: /users

    Data returned: An array of user objects.

    
[
  {
    "_id": "68cd7e3ed8540b738fcebea4",
    "username": "LesterBangs",
    "email": "lester.bangs@example.com",
    "birth_date": "1948-12-14T00:00:00.000Z",
    "favorites": ["68cd8259d8540b738fcebead", "68cd8259d8540b738fcebeac"]
  },
  ...
]

  • Allow new users to register

    Description: Registers a new user account with a provided username, password, email, and birth date.

    Method Type: POST

    Endpoint URL: /users

    Request Body: A JSON object with the user's details.

    
{
  "username": "JaneDoe",
  "password": "password123",
  "email": "jane.doe@example.com",
  "birth_date": "1990-01-01"
}

    Data returned: The newly created user object.

    
{
  "username": "JaneDoe",
  "password": "password123",
  "email": "jane.doe@example.com",
  "birth_date": "1990-01-01",
  "_id": "68cd7e3ed8540b738fcebea9"
}

  • Allow users to update their user info

    Description: Updates an existing user's information (username, password, email, or birth date).

    Method Type: PATCH

    Endpoint URL: /users/{username}

    Path Parameters: username: The current username of the user to update.

    Request Body: A JSON object containing the fields to update.

    
{
  "newUsername": "JaneDoeNew",
  "newEmail": "jane.new@example.com"
}

    Data returned: The updated user object.

    
{
  "username": "JaneDoeNew",
  "password": "password123",
  "email": "jane.new@example.com",
  "birth_date": "1990-01-01",
  "_id": "68cd7e3ed8540b738fcebea9"
}

  • Allow users to add a movie to their list of favorites

    Description: Adds a movie to a user's list of favorite movies by title. The movie is added using its `_id` to the `favorites` array.

    Method Type: PATCH

    Endpoint URL: /users/{username}/{movieTitle}

    Path Parameters: username: The username of the account. movieTitle: The title of the movie.

    Data returned: The updated user object, including the new favorite movie's `_id` in the `favorites` array.

    
{
  "_id": "68cd7e3ed8540b738fcebea4",
  "username": "LesterBangs",
  "password": "R@w_P0wer!",
  "email": "lester.bangs@example.com",
  "birth_date": "1948-12-14T00:00:00.000Z",
  "favorites": ["68cd8259d8540b738fcebead", "68cd8259d8540b738fcebeac", "68cd8259d8540b738fcebeaf"]
}

  • Allow users to remove a movie from their list of favorites

    Description: Removes a movie from a user's list of favorite movies by title.

    Method Type: DELETE

    Endpoint URL: /users/{username}/{movieTitle}

    Path Parameters: username: The username of the account. movieTitle: The title of the movie to remove.

    Data returned: The updated user object with the movie removed from the `favorites` array.

    
{
  "_id": "68cd7e3ed8540b738fcebea4",
  "username": "LesterBangs",
  "password": "R@w_P0wer!",
  "email": "lester.bangs@example.com",
  "birth_date": "1948-12-14T00:00:00.000Z",
  "favorites": ["68cd8259d8540b738fcebead", "68cd8259d8540b738fcebeac"]
}

  • Allow existing users to deregister

    Description: Deregisters (deletes) a user from the database.

    Method Type: DELETE

    Endpoint URL: /users/{username}

    Path Parameters: username: The username to deregister.

    Data returned: A string confirming the deletion.

    
"User JaneDoeNew was deregistered"

Optional Features

  • Allow users to see which actors star in which movies

    Description: Returns a list of all actors starring in a specific movie.

    Method Type: GET

    Endpoint URL: /movies/{title}/starring

    Path Parameters: title: The title of the movie.

    Data returned: An array of actors (strings).

    
["Harvey Keitel", "Tim Roth", "Chris Penn", "Steve Buscemi", "Lawrence Tierney", "Michael Madsen", "Quentin Tarantino", "Edward Bunker"]

  • Allow users to view information about different actors

    Description: Returns information about a specific actor.

    Method Type: GET

    Endpoint URL: /movies/actors/{actorName}

    Path Parameters: actorName: The name of the actor.

    Data returned: A success message (logic not yet implemented).

    
"Sucessful GET request returning info about actor"

Technical Requirements

  • The API must be a Node.js and Express application.
  • The API must use REST architecture, with URL endpoints corresponding to the data operations listed above.
  • The API must use at least three middleware modules, such as the body-parser package for reading data from requests and morgan for logging.
  • The API must use a “package.json” file.
  • The database must be built using MongoDB.
  • The business logic must be modeled with Mongoose.
  • The API must provide movie information in JSON format.
  • The JavaScript code must be error-free.
  • The API must be tested in Postman.
  • The API must include user authentication and authorization code.
  • The API must include data validation logic.
  • The API must meet data security regulations.
  • The API source code must be deployed to a publicly accessible platform like GitHub.
  • The API must be deployed to Heroku.
